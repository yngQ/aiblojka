# AiBlojka

**AI-генератор обложек для видео**

Product Requirements Document | Версия 1.3

---

## Обзор продукта

- **Название:** AiBlojka
- **Тип:** PWA (Flutter Web)
- **Цель:** Генерация обложек для видео с помощью AI
- **Целевые платформы:** YouTube, TikTok, YouTube Shorts, Instagram Reels
- **Аудитория:** Личный проект
- **Язык интерфейса:** Русский (архитектура готова к добавлению новых языков)

AiBlojka — одностраничное PWA без авторизации, которое позволяет пользователю сгенерировать обложку для видео, выбрав формат, опционально стиль и описав желаемый результат. Можно загрузить референс-изображение. Результат доступен для превью и скачивания. Приложение устанавливается на домашний экран и работает как нативное.

## Архитектура

**Поток данных:**

```
Flutter Web (GitHub Pages)  →  Cloudflare Worker (прокси)  →  Workers AI (FLUX.2 Klein 4B)
```

**Конфигурация и аналитика:**

```
Firebase Remote Config + Analytics  →  Flutter Web
```

### Безопасность

- Workers AI биндинг настроен на стороне Cloudflare, недоступен клиенту
- Cloudflare Worker проверяет Origin-заголовок запроса
- CORS настроен только на домен приложения (`https://yngq.github.io`)
- Worker поддерживает base64-изображения (референсы, модель пока не использует)

### Firebase Remote Config

- Шаблоны промптов по формату и стилю
- Конфиги платформ (размеры, названия)
- Список стилей
- Kill switch (generation_enabled) для аварийного отключения
- Обновление промптов без пересборки приложения

## Пользовательский сценарий

Приложение состоит из одного экрана. Пользовательский сценарий линейный:

1. **Выбор формата** — длинные видео (YouTube) или короткие видео (TikTok, Shorts, Reels)
2. **Выбор стиля** — опционально: gaming, влог, обучение, бизнес, развлечения
3. **Описание обложки** — свободное текстовое поле, пользователь описывает что хочет получить
4. **Загрузка референса** — опционально, изображение до 10 МБ (JPEG, PNG, WebP), конвертируется в base64
5. **Генерация** — отправка запроса через Cloudflare Worker к Workers AI
6. **Превью и скачивание** — просмотр результата, повторная генерация или сохранение

## Структура промптов

Промпты хранятся в Firebase Remote Config как JSON. Шаблон содержит плейсхолдеры, которые заменяются данными пользователя.

### Структура шаблона

```
{base_instruction} + {format_instruction} + [{style_instruction}] + {user_prompt} + {quality_suffix}
```

- **base_instruction** — общие правила генерации (что это обложка, не добавлять лишний текст)
- **format_instruction** — размер и ориентация (1920×1080 или 1080×1920)
- **style_instruction** — опционально, визуальный стиль (gaming, влог, обучение, бизнес, развлечения)
- **user_prompt** — свободное описание пользователя → {prompt}
- **quality_suffix** — технические параметры (высокое разрешение, без артефактов)

Конкретные промпты будут разработаны на этапе реализации.

## Этапы реализации

### Этап 1 — Cloudflare Worker

- Создание Worker через Cloudflare Dashboard
- Прокси-функция: принимает запрос от клиента, вызывает Workers AI (FLUX.2 Klein 4B)
- Workers AI биндинг (`AI`) в настройках Worker — внешний ключ не нужен
- Проверка Origin (`https://yngq.github.io`), CORS-заголовки
- Поддержка больших тел запросов (base64-референсы)
- Бесплатный тариф: 100,000 запросов/день

### Этап 2 — Firebase Remote Config + Analytics

- Подключение Firebase к Flutter Web
- Структура конфига: prompt_templates, platform_configs, styles, app_settings
- Шаблоны промптов с плейсхолдером {prompt}
- Kill switch: generation_enabled
- Обновление промптов без передеплоя приложения
- Firebase Analytics: события генерации, скачивания, ошибок

### Этап 3 — Сервисный слой Flutter

- Riverpod (с кодогенерацией) для state management
- RemoteConfigService — загрузка и кеширование конфигов
- PromptBuilder — сборка промпта из шаблона + пользовательского описания
- GenerationService — POST-запрос к Worker, парсинг base64-картинки
- Загрузка референс-файлов: валидация (до 10 МБ, JPEG/PNG/WebP), конвертация в base64, отправка как inlineData
- Обработка ошибок: 429 (лимит), safety block, сеть
- Локализация: строки в arb-файлах, готовность к мультиязычности

### Этап 4 — UI

- Один экран, линейный флоу
- Тёмная тема Solar Amber, glow-эффект на акцентных элементах
- Выбор формата (длинные / короткие видео)
- Выбор стиля (опционально)
- Поле ввода: свободное описание желаемой обложки
- Загрузка референс-изображения (опционально, до 10 МБ, JPEG/PNG/WebP)
- Кнопка генерации
- Превью результата + скачивание

### Этап 5 — Полировка и PWA

- Лоудер во время генерации
- Понятные сообщения об ошибках
- PWA: манифест, иконки, Service Worker, возможность установки на домашний экран
- Офлайн: UI-шелл кешируется, при попытке генерации без сети — сообщение "Требуется подключение к интернету"
- Адаптивный дизайн под десктоп и мобильные устройства
- Возможность отредактировать своё описание перед повторной генерацией (системная часть промпта скрыта)

## Ограничения и лимиты

| Параметр | Значение |
|----------|----------|
| Нейронов в день (Workers AI free tier) | 10,000 (~91 изображение) |
| Запросов в день (Cloudflare) | 100,000 |
| Макс. размер референс-файла | 10 МБ (JPEG, PNG, WebP) |
| Сброс лимитов Workers AI | Полночь UTC |
| Оплата | Не требуется (биллинг не подключен) |
| Поведение при превышении | HTTP 429 — запросы блокируются, деньги не списываются |

---

## Обработка ошибок

| Код | Причина | Сообщение пользователю |
|-----|---------|----------------------|
| 429 | Дневной лимит исчерпан | Лимит генераций исчерпан, попробуйте завтра |
| 400 | Safety-фильтр | Контент заблокирован фильтром, измените запрос |
| 5xx | Ошибка сервера | Ошибка сервера, попробуйте позже |
| Сеть | Нет соединения | Проверьте подключение к интернету |
| Файл | Превышен размер или неверный формат | Файл должен быть JPEG, PNG или WebP, до 10 МБ |

## Риски

| Риск | Митигация |
|------|-----------|
| FLUX.2 Klein 4B может быть заменён новой версией модели | Обновить `WORKERS_AI_MODEL` в `worker.js`, протестировать API |
| Cloudflare может сократить бесплатные Neurons | Kill switch в Remote Config; переход на Workers Paid план |
| Злоупотребление эндпоинтом Worker | Проверка Origin; rate limiting по IP |
